# Knowledge Base Deep Reading Skill

> 基于转录文稿的精读知识库技能

## 用途

对 `/path/to/transcripts/` 下的所有转录文稿进行**精读提炼**，而不是简单摘要。输出可直接执行的方法论。

## 知识库路径

```
/path/to/transcripts/
├── 爱戏剧的柏拉图-视频转录/
│   ├── 音频文件/          # 319 个音频
│   └── 转录结果/          # 318 个 txt 文稿
├── 恋爱脱单实战课：从心动到确定关系/
│   ├── 提取音频/          # 19 个音频
│   └── 转录结果/          # 19 个 txt 文稿
├── 一门给年轻人的恋爱成长课/
│   ├── 提取音频/          # 37 个音频
│   └── 转录结果/          # 37 个 txt 文稿
```

**总计：374 篇文稿，全部为 .txt 纯文本**

## 使用方式

### 方式一：按主题精读（推荐）

用户提出一个具体问题或主题，例如：
- "帮我精读所有关于'破冰/第一印象'的文稿"
- "微信聊天技巧有哪些？把所有相关的方法整理出来"
- "怎么处理冲突和吵架？从文稿里提炼完整方法论"

AI 执行以下流程：
1. **检索**：扫描所有 374 篇文稿，找出与主题相关的内容
2. **精读**：逐篇深度阅读，不是关键词匹配，是理解实际在讲什么
3. **提炼**：每篇只留三样东西：核心观点、可执行步骤、关键话术
4. **整合**：同一主题下的方法去重、合并、排序，形成完整体系
5. **输出**：按 SKILL_DEEP_READ_TEMPLATE 格式生成文档

### 方式二：单篇精读

用户提供具体文件名，例如：
- "精读这篇：电子榨菜004. 4种低情商行为，让人秒变油腻.txt"

AI 输出该篇的完整精读卡片。

### 方式三：批量主题归类

用户要求："把所有 374 篇按主题归类"

AI 执行：
1. 快速浏览所有文稿标题和开头
2. 建立主题分类体系
3. 输出分类目录树
4. 用户确认后，再逐类精读

## 精读标准（严格执行）

### ❌ 不要做的事
- 不要批量生成 3 句话摘要（那是索引，不是精读）
- 不要原文复制大段内容
- 不要说"这篇文章讲了..."这种废话
- 不要 AI 味的总结和升华
- 不要每篇孤立输出

### ✅ 必须做的事
- 真正理解文稿在说什么（不是表面关键词匹配）
- 提炼底层逻辑，去掉故事和废话
- 把同一个主题下不同文章的方法串起来
- 输出结构化的可执行方法论
- 标注每个方法的出处（方便回溯原文）

```markdown

**核心观点**：一句话概括
```

## 执行命令

当用户要求精读某个主题时，执行：

```bash
# 1. 检索相关文稿
grep -rl "关键词" /path/to/transcripts/*/转录结果/

# 2. 逐篇精读
cat "/path/to/file.txt" | head -50  # 先看开头判断主题相关性

# 3. 输出精读结果
```

## 注意事项

1. **一次只做一个主题**，不要一次性精读全部 374 篇
2. **优先做用户当前最需要的主题**（根据对话上下文判断）
3. **精读后必须可执行**，输出的方法要具体到可以直接用
4. **标注出处**，每个方法都要注明来自哪篇文稿
5. **不要过度解读**，忠于原文，不添加原文没有的内容

## 与 AI 问答系统的区别

| | 知识库问答（RAG） | 精读技能（本技能） |
|---|---|---|
| 目的 | 快速回答问题 | 深度提炼方法论 |
| 输出 | 一段回答 | 完整的主题手册 |
| 深度 | 检索最相关内容 | 逐篇精读+交叉整合 |
| 使用场景 | 遇到具体问题时 | 系统学习某个主题时 |
